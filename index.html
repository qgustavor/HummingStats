Hummingbird allows users show the world what waifus and husbandos they like. The website also have a API that allow getting this values for each user. How about making a rank of those?

We don't have a direct connection with Hummingbird database, so the data isn't real time and is slowly being updated from the API.

When romanized names are detected (using Hepburn) then the Japanese name order is used. I know that Hummingbird, MyAnimeList and Wikia many others uses Western order, but as when searching for those the Japanese order returns more results, that's the canonical order.

Avatars from Hummingbird, using cropped MyAnimeList images as a fallback (case when the only available avatars in Hummingbird are low res ones or if other problem happen).

Some links in waifu and husbando names can be broken. I think that's because the URL of the anime it's linking changed and the API returned the old one from when the user set up his waifu or husbando.

In life spent charts counts below 4 aren't shown, to avoid showing situations like people that are watching anime 24/7 since the Roman Empire fall (anime don't even existed in this epoch: time travel?).

Sadly Google still don't implemented material charts for pie charts and logarithmic scaling don't work in column charts, so the classic and old charts had to be used.

Because a bug in Hummingbird code is possible to a user have something which something which isn't a waifu or a husbando (the "other" in the first graph in preferences). Also, as husbandos aren't that popular data from those are quite limited, so many of those ends tied. To solve this problem watching time is used as a tie breaker.
